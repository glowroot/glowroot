How Slow / Error trace filters work

Filters apply only after Refresh. They narrow traces on this chart and the tab count; they do not change average / percentiles / throughput.

  • Response time — duration (ms): greater / less / between.
  • Headline/URL — stored headline (often URL or txn text). Comparators: begins, equals, ends, contains, does not contain. Not full query-string search unless it is in the headline.
  • Error message — error text (same comparators as Headline/URL).
  • User — user on the trace (servlet plugin user attribute).
  • Attribute name / value — custom / session attributes. Dropdown only if names exist for this transaction type.
  • Result limit — max chart points (slowest matches). Over limit → warning; tab count still shows matching total.

Clear criteria resets all filters and refreshes.
